# 1方案原理

## 1.1 基于 YOLO11 的水果检测模型改进方案

本文以原始 YOLO11 网络为基础，设计四种对比模型，分别为原始 YOLO11 网络、引入度量学习的 YOLO11 网络、度量学习 + DCNv2 可变形卷积的 YOLO11 网络、度量学习 + 多原型检测头的 YOLO11 网络。原报告中对于水果检测的改进方案是引入 SGE 模块，但该模块的功能已经在原 YOLO11 网络中 CSP 块实现，因此不考虑。各模型改进原理如下：

### 1.1.1 原始 YOLO11 基础网络原理

YOLO11 是 Ultralytics 团队推出的新一代轻量化目标检测算法，延续 YOLO 系列端到端的检测架构，整体分为主干网络、颈部网络、检测头三部分，各模块结构相较于前代版本完成了轻量化与精度的双重升级。其中主干网络核心由 C2PSA 模块、C3K2 模块以及 SPPF 空间金字塔池化模块构成，C2PSA 与 C3K2 轻量化卷积模块可高效逐级提取图像浅层纹理、边缘特征与深层语义特征，SPPF 模块负责拓宽感受野，保留多尺度全局特征；颈部网络以 PANet 特征融合结构为主体，搭载 C3K2 模块与 CBS 卷积模块完成多层特征拼接与维度校准，实现高低层特征的高效融合，强化对不同尺寸水果目标的适配能力；检测头采用解耦头结构，将分类任务与回归任务分离，有效避免任务干扰，同步提升检测精度与推理速度。

YOLO11 具备参数量小、推理速度快、部署门槛低的优势，适配嵌入式设备与实时检测场景，但针对水果细粒度区分、不规则形变水果、复杂背景干扰场景，特征区分能力仍存在不足。

### 1.1.2 引入度量学习的 YOLO11 改进原理

原始 YOLO11 检测头仅依靠单一分类分支、固定类别通道与常规 BCE、DFL、Bounding Box 基础损失函数完成目标识别，特征表征维度单一，对外观、色彩高度相似的水果品类细粒度区分能力极差，极易出现类别混淆、误检问题。为解决该缺陷，本文对 YOLO11 原生 Detect 检测头与损失计算逻辑进行改造。

在网络结构层面，在检测头内部新增独立 cv4 嵌入卷积分支，对多尺度特征图逐级压缩、编码，输出固定维度的高维特征嵌入向量，为度量学习提供特征输入；

 在损失函数层面，重构原生 v8DetectionLoss 损失类，新增度量中心损失（Embedding Loss），构建检测定位损失、分类损失、DFL 分布损失、特征度量损失的多任务联合优化体系。

训练阶段，模型通过锚点特征嵌入向量与可学习类别中心向量做 MSE 距离约束，拉大不同水果类别特征间距、缩小同类特征相似度，从特征空间底层优化模型细粒度表征能力，彻底解决相似水果混淆、错检的核心问题。推理阶段仅保留检测与分类分支，嵌入分支不参与计算，在不增加推理开销的前提下理论上可以大幅提升模型分类精度。

### 1.1.3 度量学习 + DCNv2 可变形卷积改进原理

标准卷积核采用固定矩形采样窗口，仅能提取规则图像特征，而自然场景下的水果存在遮挡、形变、角度偏移等问题，标准卷积难以适配不规则目标特征提取。DCNv2 可变形卷积在标准卷积基础上，增加偏移量学习与权重修正机制，可根据目标形态自适应调整卷积采样位置与采样权重，精准提取形变、遮挡水果的有效特征。

本方案在度量学习优化特征空间的基础上，将网络颈部的 P4、P5 的 C3K2 块中的标准卷积替换为 DCNv2 可变形卷积，兼顾特征区分度与形变特征提取能力，理论上有利于解决水果形变、局部遮挡导致的漏检、精度下降问题。由于数据集中水果大多数为中大型目标，因此只选择替换了 P4、P5 层级 C3K2 模块。

### 1.1.4 度量学习 + 多原型检测头改进原理

仅引入度量学习虽能优化类别特征可分性，但传统单原型分类机制仅有单一组特征模板，无法适配同一水果品类的多形态差异（完整果实、切开果实、遮挡形变、不同光照形态），模型泛化能力存在明显瓶颈。因此本文在度量学习嵌入分支的基础上，进一步重构 Detect 检测头为多原型检测结构，并适配自定义多原型损失计算逻辑，实现度量学习与多原型分类机制的深度融合。

网络结构上，新增`num_prototypes`子原型参数，将原始单类别输出通道`nc`改造为`nc_out=nc×num_prototypes`多原型融合通道，为每一类水果配置多组专属特征模板，适配同类水果的不同形态特征。

损失逻辑上，改造 v8DetectionLoss 的 TAL 任务对齐匹配机制，先对多原型预测分数取最大值完成前后帧目标匹配，再通过 “赢家通吃” 机制动态分配各原型训练标签，对最优原型梯度更新、其余原型做背景惩罚，同时结合类别多原型中心向量计算度量损失，同步约束类间特征区分度与类内多形态特征适配性。推理时对多原型得分做最大值融合，输出最优分类结果，理论上有利于解决单一模板适配性差、异形水果漏检误检问题。

## 2.1 水果检测模型训练（四种对比模型）

### 2.1.1 实验环境与数据集配置

本次模型训练基于 Python 深度学习框架搭建，配置 CUDA 加速环境，硬件设备为 NVIDIA 独立显卡，保障模型训练与推理效率。数据集采用自己建立的水果检测数据集，包含苹果、西瓜、橙子、香蕉、草莓、猕猴桃、菠萝、榴莲、火龙果水果图像，在正常光照场景添加光照变化、图像平移与旋转变化的图像变换，共计 1762 张图像，按照 8:1:1 比例划分为训练集、验证集、测试集。

训练前完成数据集预处理，包括图像尺寸统一、归一化、随机裁剪、翻转、色域变换等数据增强操作，扩充样本多样性，防止模型过拟合。同时完成标签校验、类别筛选，保证数据集标注精准度。

### 2.1.2 通用训练参数设置

四种模型均采用统一超参数配置，保证实验对比的公平性。

- 输入图像尺寸：640×640
- BatchSize：24
- 迭代轮次 Epochs：170
- 初始学习率：0.001
- 学习率策略：余弦退火衰减
- 优化器：Adam
- 权重衰减系数：5e-4
- 早停策略：验证集精度连续 30 轮无提升终止训练，保存最优模型权重

### 2.1.3 各模型差异化训练设置

#### （1）原始 YOLO11 模型

采用网络默认损失函数，包含分类损失、定位损失、置信度损失，无额外模块引入，全程基础参数迭代训练，保存最优权重模型作为对照组。

#### （2）度量学习改进 YOLO11 模型

在 YOLO11 原生检测损失（定位损失、分类 BCE 损失、DFL 分布损失）基础上，接入自定义改进损失函数，新增多原型度量嵌入损失。训练过程中通过 cv4 分支提取特征嵌入向量，匹配各类别对应的多组可学习原型中心，利用 MSE 损失约束特征聚类效果，通过`embed_loss_weight`权重参数平衡检测主损失与度量辅助损失，同步优化模型定位精度与细粒度特征区分能力，解决相似水果品类混淆问题。

#### （3）度量学习 + DCNv2 可变形卷积模型

替换网络中 1/3 的标准卷积为 DCNv2 可变形卷积，开启偏移量参数训练，结合度量学习分支，同步优化形变特征提取与细粒度特征区分能力，适配复杂场景水果检测。由于数据集中水果大多数为中大型目标，因此只选择替换了 P4、P5 的 C3K2 块。

#### （4）度量学习 + 多原型检测头模型

启用多原型通道拆分与动态标签分配机制，训练时对`nc×num_prototypes`维度的预测分数进行原型维度拆解，通过 argmax 筛选单网格最优原型，配合改良 TAL 对齐算法完成目标匹配与梯度反向传播。同时结合多原型中心度量损失，既保证不同水果类别特征完全聚类分离，又适配同一水果的多形态特征差异，大幅提升复杂场景下水果分类、检测的泛化能力与稳定性。

## 3.1 水果检测模型测试与结果分析

本次测试统一采用验证集数据，使用 Nvidia 4060 Laptop 显卡，`ImageSize=640*640`，`Batch=24`相同设置下，以精确率 Precision、召回率 Recall、平均精度均值 mAP@0.5、推理速度 FPS 作为核心评价指标，对四种模型进行全方位性能对比测试。

根据测试结果可以看出度量学习 + 多原型检测的召回率是最高的，模型漏检抑制效果最优；精确率相对偏低，主要源于模型检出更多真实正样本，而数据集存在标注缺失、标注不全的问题，属于数据集质量导致的假性精度下降。仅加入度量学习的模型推理速度与原始 YOLO11 基本持平，验证了 “训练辅助、推理无开销” 结构的优越性。多原型检测模型运行速度受画面目标数量影响较大，帧率波动明显。引入 DCNv2 的模型因卷积偏移量计算引入额外计算量，帧率下降最为显著。

>  根据图标可以看到度量学习 + 多原型检测的召回率是最高的，至于精确率较低主要原因是模型识别到更多正样本，而数据集没有标注（数据集质量差）。只加入度量学习的模型由于推理时嵌入空间表达分支不运行，因此运行速度和原 YOLO11 模型相同。而增加了多原型检测的模型运行速度受图片中物体数量影响较大，因此运行速度波动较大。召回率高，说明模型改进后漏检的情况减少了。引入可变形卷积的模型由于在计算卷积时增加了偏移量，因此帧率下降明显。

### 3.1.1 原始 YOLO11 模型测试结果

原始 YOLO11 模型检测效果较好，mAP@0.5 为 89.1%，推理 FPS 可达 142 帧 /s，实时性优异。通过混淆矩阵分析可得，模型对苹果、橙子识别精度偏低，核心原因是两类水果同时存在完整形态与切开形态，类内视觉差异极大，单一特征模板难以收敛；香蕉识别效果较差来源于长条形态导致标注框容易引入背景噪声，特征纯净度低。整体模型对类内异构水果适应性差，漏检与误检问题突出。

### 3.1.2 度量学习改进模型测试结果

引入度量学习后，模型 mAP@0.5 为 86.5%，召回率明显提升，漏检数量减少，细粒度特征区分能力得到优化。但整体 mAP 出现小幅下降，原因在于本文采用的 MSE 中心度量损失对类内差异极度敏感，数据集完整水果样本占比远大于切开水果，模型特征中心偏向完整水果样本，导致少量切开水果特征被强制拉扯扭曲，特征空间出现混叠，反而干扰部分类别分类效果。虽然缓解了相似水果误检，但无法解决 “完整与切开水果极端异构” 的本质矛盾。

### 3.1.3 度量学习 + DCNv2 可变形卷积模型测试结果

该改进方案 mAP@0.5 为 85.9%，帧率下降明显。DCNv2 可变形卷积能够有效适配水果形变、角度变化与局部遮挡特征提取，但实验结果表明，限制本数据集水果检测精度的核心瓶颈不在于特征提取能力，而在于同类水果形态差异过大导致的分类歧义，因此可变形卷积带来的增益有限，无法从根本上解决类内异构问题。

### 3.1.4 度量学习 + 多原型检测头模型测试结果

融合多原型检测头后，模型 mAP@0.5 回升至 88.7%，召回率为四组模型最优，对猕猴桃、橙子等形态多变水果的识别效果提升明显。多原型机制通过双模板适配完整与切开两种形态，有效降低漏检率。但该方案整体提升效果有限，存在明显缺陷：多原型仍然假设每种子类样本特征紧凑聚拢，而切开水果形态千差万别，有限的原型数量无法覆盖全部形态；同时训练过程中在线原型分配不稳定、样本数量不均衡，导致部分原型中心更新滞后、特征空间分裂，训练噪声较大，限制了模型性能进一步提升。

## 4 研究总结

在水果检测任务中，本文针对水果检测中同类水果完整形态与切开形态视觉差异极大、细粒度混淆严重的核心难点，基于 YOLO11 轻量化网络设计四组递进改进模型，分别引入纯度量学习、度量学习 + DCNv2、度量学习 + 多原型检测头三种改进策略，并通过统一实验环境与超参数完成公平对比。实验结果表明：

1. 原始 YOLO11 推理速度最快，但对形态异构水果收敛困难、识别精度偏低；
2. 单纯度量学习能够优化特征聚类、降低漏检，但受限于 MSE 中心损失的强制聚拢特性，对极端异构样本适应性差，mAP 小幅下降；
3. DCNv2 可变形卷积能够提升形变特征提取能力，但无法解决类别级的形态歧义问题，整体增益有限；
4. 度量学习 + 多原型检测头通过双原型适配完整与切开两类形态，召回率显著提升，综合性能最优，是四组模型中最适合本水果数据集的改进方案。

同时本文通过大量实验复盘，探明了多原型 + 度量学习方案效果未达预期、提升幅度有限的深层机理：该方法本质仍属于 “聚类约束思路”，强行要求类内样本特征紧凑聚拢，而完整水果与切开水果属于类内极端异构，无法通过简单多原型聚类彻底解耦；同时在线原型分配不稳定、样本数量不均衡、MSE 损失特征坍缩、轻量化模型表征能力有限等问题叠加，导致训练存在噪声、特征空间分裂，限制了模型性能上限。

## 5.2 现存不足

1. **度量学习机制存在固有缺陷**：本文采用的 MSE 中心损失对类内差异过于敏感，容易迫使极端异构样本特征坍缩、特征混叠，无法容忍完整水果与切开水果的巨大视觉差异，不仅难以提升精度，甚至会轻微干扰模型收敛。
2. **多原型检测机制训练不稳定**：训练阶段采用在线原型分配方式，模型前期特征未收敛，容易出现原型分配错误，造成中心来回震荡、梯度信号矛盾；同时数据集完整水果与切开水果样本不均衡，导致子原型更新不足、形成 “死中心”，特征表达能力弱。
3. **模型结构解耦能力不足**：仅依靠检测头嵌入分支与少量颈部 DCN 卷积，YOLO11n 轻量化主干的特征解耦能力有限，无法从根源上区分 “同类不同形态” 的特征差异，模块改进治标不治本。
4. **数据集质量与样本覆盖不足**：自制数据集总量有限、切开类样本偏少、存在标注缺失与模糊样本，导致模型训练存在偏置，同时模型精确率指标受标注质量影响出现假性降低。

## 6.3 未来改进方向

针对本次实验暴露的核心问题，结合多原型度量学习失效机理，后续可从损失函数、模型结构、训练策略、数据增强四个维度进行系统性优化：

1. **替换度量损失函数，摒弃强制聚类思路**：放弃容易造成特征坍缩的 MSE Center Loss，改用 ArcFace 角度间隔损失、CosFace 或多原型对比损失，通过角度边际约束拉大类别间距，天然容忍类内形态多样性，不再强制异构样本强行聚拢。
2. **重构形态解耦网络结构**：放弃单一检测头聚类优化思路，采用双分支解耦检测头，分别负责完整水果与切开水果特征识别，共享主干与颈部特征，从结构层面解决形态异构问题；同时引入动态卷积 DY-CNN 替代部分 DCNv2，进一步提升模型自适应形变能力。
3. **层次化多原型特征优化**：基于特征金字塔特性，让浅层 P3 特征负责切开水果局部纹理识别、深层 P5 特征负责完整水果整体形状识别，分层施加原型约束，避免跨尺度特征拉扯冲突。
4. **强化数据增强与样本均衡**：采用 Mosaic、MixUp、Copy-Paste 复合增强策略，主动合成完整与切开水果共存场景；对稀少的切开样本进行过采样与针对性增强，平衡两类子原型的训练梯度，解决死中心问题。
